Building ROS 2 Packages with Python

In this section, you will learn the fundamental process of creating and building a ROS 2 package using Python. ROS 2 packages are the primary way to organize your code, making it modular, reusable, and easy to share.

Learning Objectives​

After completing this section, you will be able to:

  • Create a new ROS 2 Python package.
  • Understand the directory structure of a typical ROS 2 package.
  • Configure the package.xml and setup.py files.
  • Build your package using colcon.
  • Run a Python node from your custom package.

The Anatomy of a ROS 2 Package​

A ROS 2 package is a directory containing your source code, launch files, configuration, and a package.xml manifest file. This manifest file contains metadata about the package, such as its name, version, dependencies, and author information. For Python packages, it also includes a setup.py file to manage installation.

[Diagram: A high-level view of a ROS 2 workspace containing multiple packages, showing how they are isolated but can communicate.]

Step 1: Creating a ROS 2 Package​

The first step is to create a new package. ROS 2 provides a command-line tool to automate this process. Navigate to your workspace's src directory and run the following command:

ros2 pkg create --build-type ament_python --node-name my_python_node my_python_pkg

This command creates a new directory named my_python_pkg with the basic structure for a Python-based ROS 2 package. It also creates a simple executable node named my_python_node.

Let's examine the files and directories that were just created:

  • package.xml: This is the manifest file. It contains meta-information about the package.
  • setup.py: This is the build script for Python packages. It describes how to install the package and its modules.
  • setup.cfg: A configuration file for setuptools.
  • my_python_pkg/: A Python sub-directory that contains your package's modules. The my_python_node.py file holds the source code for your ROS 2 node.
  • resource/: This directory is used to place a marker file so that ROS 2 can find your package.

Step 3: Configuring package.xml​

The package.xml file describes your package's dependencies and metadata. A minimal example looks like this:

<?xml version="1.0"?>
<package format="3">
  <name>my_python_pkg</name>
  <version>0.0.0</version>
  <description>A simple Python package for ROS 2</description>
  <maintainer email="user@example.com">Your Name</maintainer>
  <license>Apache License 2.0</license>

  <depend>rclpy</depend>

  <test_depend>ament_copyright</test_depend>
  <test_depend>ament_flake8</test_depend>
  <test_depend>ament_pep257</test_depend>
  <test_depend>python3-pytest</test_depend>

  <export>
    <build_type>ament_python</build_type>
  </export>
</package>

Key tags to note are:

  • <name>: The name of your package.
  • <version>: The version number.
  • <description>: A brief summary of what the package does.
  • <maintainer>: The person responsible for the package.
  • <license>: The software license for your code.
  • <depend>: These tags declare dependencies on other ROS 2 packages. rclpy is the ROS 2 client library for Python.

Step 4: Configuring setup.py​

The setup.py file is essential for Python packages. It tells colcon how to build and install your code, including where to find your executable nodes.

from setuptools import setup

package_name = 'my_python_pkg'

setup(
    name=package_name,
    version='0.0.0',
    packages=[package_name],
    data_files=[
        ('share/ament_index/resource_index/packages',
            ['resource/' + package_name]),
        ('share/' + package_name, ['package.xml']),
    ],
    install_requires=['setuptools'],
    zip_safe=True,
    maintainer='Your Name',
    maintainer_email='user@example.com',
    description='A simple Python package for ROS 2',
    license='Apache License 2.0',
    tests_require=['pytest'],
    entry_points={
        'console_scripts': [
            'my_node = my_python_pkg.my_python_node:main'
        ],
    },
)

The most important part here is the entry_points dictionary. It maps the name of your executable (my_node) to the location of the main function in your Python script. This allows you to run your node using ros2 run.

Step 5: Building the Package​

Now that your package is set up, you can build it. Navigate to the root of your ROS 2 workspace and run colcon build:

colcon build --packages-select my_python_pkg

This command will compile your package and install the necessary files into the install directory in your workspace. If the build is successful, you will see a confirmation message.

[Illustration: A terminal window showing the output of a successful colcon build.]

Step 6: Sourcing and Running the Node​

Before you can run your new node, you need to source your workspace's setup file. This adds your package to the ROS 2 environment, allowing it to be found.

# In Linux/macOS
source install/setup.bash

# In Windows
call install/setup.bat

Finally, you can run your node using ros2 run:

ros2 run my_python_pkg my_node

If everything is configured correctly, your Python node will start executing.

[Example: The expected output of the my_python_node when it runs, such as a "Hello, ROS 2!" message being printed to the console.]

Summary​

You have successfully created, configured, built, and run a ROS 2 package using Python. This foundational skill is crucial for developing more complex robotic applications. By organizing your code into packages, you embrace the modularity and reusability that make ROS 2 a powerful framework.
